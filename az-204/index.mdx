**Bu kitab kimlər üçündür**

Bu kitab Microsoft Azure təcrübəsi olan peşəkar developerlərə AZ-204: Microsoft Azure imtahanına hazırlaşmağa və imtahan vermək istəyən developerlər üçün nəzərdə tutulub.

**Bu kitab nəyi əhatə edir**

**1-ci fəsil – Azure və Bulud Əsasları** Bu fəsildə bulud texnologiyalarının əsas anlayışları və Azure-un vacib məqamları sadə dildə izah olunur. Oxucu kitabın davamını rahat başa düşmək üçün lazım olan ilkin bilikləri əldə edir.

**2-ci fəsil – IaaS Həllərinin Qurulması** Bu fəsil əvvəlki mövzuların davamıdır və Infrastructure as a Service anlayışını izah edir. Virtual maşınlar, ARM şablonları, eləcə də konteynerlər və onların Azure-da istifadəsi nümunələrlə göstərilir.

**3-cü fəsil – Azure App Service ilə Web Tətbiqlər** Burada Azure-da ən çox istifadə olunan xidmətlərdən biri olan Azure App Service izah edilir. App Service-in nə olduğu, istifadəçi girişinin idarə edilməsi, təhlükəsizlik, şəbəkə imkanları, tətbiq parametrləri və loglama mövzuları sadə formada açıqlanır.

**4-cü fəsil – Azure Functions** Bu fəsil Azure Functions-a giriş edir. Funksiyaların necə işlədiyi, necə miqyaslandığı, necə yazılıb test edildiyi və necə yayımlandığı izah olunur. Həmçinin durable (vəziyyət saxlayan) funksiyalara da toxunulur.

**5-ci fəsil – Cosmos DB ilə İşləyən Həllər** Bu fəsildə Azure üzərində NoSQL verilənlər bazaları izah olunur. Azure Table Storage və Cosmos DB-nin necə işlədiyi, miqyaslanma, etibarlılıq və məlumatların qorunması kimi mövzular sadə dildə təqdim edilir. Nümunələr Azure portalı və kod vasitəsilə göstərilir.

**6-cı fəsil – Azure Blob Storage-dən İstifadə Edən Həllərin Hazırlanması** Bu fəsildə Azure Blob Storage izah olunur və onun Azure-da işləyən tətbiq və servislərdə hansı rolu oynadığı göstərilir. Blob Storage-in əsas imkanları öyrədilir və hansı hallarda web tətbiqlərin saxlanması üçün faydalı olduğu nümunələrlə açıqlanır.

**7-ci fəsil – İstifadəçi Autentifikasiyası və Avtorizasiyası** Bu fəsil Microsoft identity platformasına sadə və ətraflı girişlə başlayır. Daha sonra Microsoft Authentication Library (MSAL) ilə istifadəçi girişinin necə qurulduğu göstərilir. Ardınca Microsoft Graph tanıdılır və kod üzərindən ona sorğuların göndərilməsi izah edilir. Fəsil Azure storage resurslarına giriş üçün istifadə olunan Shared Access Signature (SAS) mövzusu ilə tamamlanır.

**8-ci fəsil – Təhlükəsiz Bulud Həllərinin Qurulması** Burada tətbiqlərin gizli məlumatlarını qorumaq üçün Azure Key Vault-dan istifadə izah olunur. Key Vault ilə autentifikasiya və icazə mexanizmləri sadə şəkildə açıqlanır. Daha sonra managed identity anlayışı, onun istifadə variantları və ən yaxşı yanaşmalar təqdim edilir. Fəsilin sonunda Azure App Configuration ilə tətbiq ayarlarının və feature flag-ların mərkəzləşdirilmiş və təhlükəsiz idarə olunması izah olunur.

**9-cu fəsil – Keşləmə və Content Delivery İnteqrasiyası** Bu fəsil dinamik məlumatların keşlənməsi üçün Azure Cache for Redis ilə başlayır. Ardınca statik məzmunun keşlənməsi üçün Azure CDN izah olunur. Keşləmə yanaşmaları, yüksək əlçatanlıq, qiymət modelləri və Azure App Service və Azure Blob Storage kimi xidmətlərlə inteqrasiya mövzularına toxunulur.

**10-cu fəsil – Metriklər və Loglar ilə Problemlərin Aşkarlanması** Burada monitorinq və izləmə mövzuları izah edilir. Application Insights ilə tətbiq xətalarının tapılması, Azure Monitor ilə logların izlənməsi və Kusto sorğuları ilə canlı dashboard və hesabatların hazırlanması sadə dildə təqdim olunur.

**11-ci fəsil – API Management-in Tətbiqi** Bu fəsil web API-lər, Swagger və Azure API Management servisinə fokuslanır. API-lərin idarə olunması və policy-lər vasitəsilə daha inkişaf etmiş konfiqurasiyaların qurulması izah edilir.

**12-ci fəsil – Hadisə Əsaslı Həllərin Hazırlanması** Bu fəsildə Azure-da mövcud olan event-based servislər tanıdılır. Böyük həcmli məlumatların qəbulu üçün Event Hubs, reaktiv sistemlər üçün Event Grid, IoT məlumatlarının izlənməsi üçün IoT Hub və sistemlərarası əlaqə üçün Azure Relay izah olunur.

**13-cü fəsil – Mesaj Əsaslı Həllərin Hazırlanması** Burada mesajlaşma yanaşmaları izah edilir. Azure Storage Queue və Azure Service Bus üzərindən mesajlaşma servisləri tanıdılır. Etibarlı mesaj ötürülməsi üçün əsas prinsiplər verilir və fəsilin sonunda DevOps anlayışına qısa giriş və praktik nümunələr təqdim olunur.

**14-cü fəsil – Sınaq İmtahan Sualları** Bu fəsil imtahana hazırlıq məqsədi daşıyır. İmtahanda çıxan sual tipləri, vaxt bölgüsü izah olunur və dərhal sınaq imtahan sualları ilə praktik məşq imkanı yaradılır.

### İstifadə olunan işarələr və yazı qaydaları

Bu kitab boyunca müxtəlif yazı qaydalarından istifadə olunub.

**Mətn daxilində kod** – mətn içində yazılan kod sözlərini, verilənlər bazası cədvəl adlarını, qovluq və fayl adlarını, fayl uzantılarını, yol adlarını, nümunə URL-ləri və istifadəçi girişlərini göstərir. Məsələn: “URL daxilində scope parametri var: scope=openid+profile+email. Bu parametr [User.Read](http://User.Read) icazəsinin istifadə etdiyi standart OIDC icazələrindən ibarətdir.”

Kod blokları aşağıdakı kimi göstərilir:

```c#
using Microsoft.Identity.Client;
const string _clientId = "<app/client ID>";
const string _tenantId = "<tenant ID>";
var app = PublicClientApplicationBuilder
```

Kodun müəyyən hissəsinə xüsusi diqqət çəkmək lazım olduqda, həmin sətirlər qalın şəkildə göstərilir:

```c#
var app = PublicClientApplicationBuilder
    .Create(_clientId)
    .WithAuthority(AzureCloudInstance.AzurePublic, _tenantId)
    .WithRedirectUri("<http://localhost>")
    .Build();
```

Komanda sətri (command line) üzərindən daxil edilən əmrlər və çıxışlar isə belə yazılır:

```.NET
$ dotnet new console -n "<app name>"
```

**Qalın yazı (Bold)** – yeni terminləri, vacib sözləri və ya ekranda gördüyünüz mətnləri göstərir. Məsələn, menyu və dialoq pəncərələrindəki sözlər qalın yazılır. Nümunə: “**Identity provider** olaraq **Microsoft**-u seçin, hər şeyi standart saxlayın və **Add** düyməsinə klikləyin.”
